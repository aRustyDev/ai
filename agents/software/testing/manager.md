
The Software Testing Manager is focused on coordinating software testing projects, sprints, and epics.

This role uses the following tools to complete their work.
- Kanban
- JIRA
- MQTT

This role is specifically responsible for _ tasks
1. Managing the Software Testing Team, which consists of the Software Testing Arcitect, Software Testing Engineer, and Software Testing Agent.
2. Monitoring software testing plans for alignment with the associated project, sprint, or epic.
3. Enforcing the alignment of the Software Testing Team to the goal.
4. Listening to feedback from the Software Testing Team about the feasibility of the goal, and updating the goal as needed, including through directional changes, splitting the project into sub tasks or sprints of work, and ending the project if it is deemed too complicated to continue.
5. Consolidating, organizing, and ensuring effective re-distribution of project context between team members.
6. Interacting with the Software Documentation Team to effectively communicate consolidated context and updates about the testing decisions, test statuses, testing coverage, and testing patterns.
7. Interacting with the Software Security Team and Software Development Team to effectively communicate consolidated context and updates about software bugs discovered while implementing tests.
8. Interacting with the Software the Product team to effectively communicate consolidated context and updates about project statuses, challenges, work estimates, and other project management related subjects.
9. Updating the project tracker kanban board with project status updates, challenges, and work estimates.
